Introduction

Nathan’s Famous Hotdog Eating Contest is a contest held every year on the 4th of July in which competitive eaters compete to eat the most amount of hotdogs in a specified amount of time. Recent competitions have been run for a total of either 10 or 12 minutes, but earlier competitions were much shorter.

This dataset looks at the winner of the competition each year, the gender (only some years have data on female winners), the year the competition took place, the total runtime of the competition that year, how many hotdogs were eaten by the winner, and variable of how many hotdogs per minute the winner ate to account for different competition runtimes.

In this module, we will look at how to use the Closeread Quarto extension to tell a story using data.

Here is an example Nathan’s Hotdog Eating Contest Closeread telling the story of the contest using data visualization. This is what we will be creating in this module.

The data for this module comes from Wikipedia, and has been cleaned into a .csv file for convenience.

Take a look at the Nathan’s Hotdog Eating Contest Wikipedia page to see the original data and familiarize yourself with the story. There are many anecdotes that explain changes in the data and you will want to have an idea of these in order to effectively communicate your storytelling.

The cleaned data set can be downloaded and imported here.

Download Data: HotdogData.csv

Data

The cleaned hotdog dataset contains 69 rows, one per competition champion, and includes variables of the year they won, gender, how many hotdogs were eaten by the winner, the run time of the competition that year, and a proportion variable of how many hotdogs were eaten per minute.

Download Data: HotdogData.csv

Variable Description
Year The year the competition took place
Sex Indicator variable of sex (1 for male, 0 for female)
Dogs How many total hotdogs were eaten by the winner
Winner Name of the competition winner
Time Time (in minutes) of the competition (not consistant year to year)
Hotdogs_per_minute Proportion of how many hotdogs per minute were eaten (to account for time difference)

Creating Your Closeread

Motivating question

In this module, we investigate the following research question:

What are some factors that contribute to how many hotdogs the winner of Nathan’s Famous Hotdog Eating Contest eats and how can we tell that story using data?

In other words, how can we use Closeread for effective data storytelling?

Prepping Workspace

In a new Quarto Document, paste the following header to create a Closeread document

---
title: Nathan's Hotdog Eating Contest
author: Your Name
format:
  closeread-html:
    cr-style:
      narrative-background-color-sidebar: "#182"
      narrative-text-color-sidebar: "#fff"
      section-background-color: "#f7f6f5"
---

The format for a Closeread header is similar to that of the typical .html document with a few alterations. Most of the aesthetics are coded for in the header.

  • narrative-background-color-sidebar changes the color of the sidebar

  • narrative-text-color-sidebar changes the color of the text on the sidebar

  • section-background-color changes the overall background color

There are many more aesthetic changes that can be made to the Closeread document. Look at the Closeread website to find more aesthetic options. Feel free to style it to your preference.

To begin, we will need to load the packages we will be using. For this module, you’ll need to load the tidyverse and ggrepel packages.

Reshaping Data

After looking at some of the information on the Wikipedia page on the Nathan’s Hotdog Eating Contest, you may have noticed that every year has a male winner but the female winners are not as consistent. Let’s start by using the indicator variable for Sex and filtering to only include male winners for our first visualization. For now we only want to look at male winners.

Try on your own to use the filter function to modify the dataset.

Click to reveal code 
hotdog_men <- hotdog_cleaned |> filter(Sex == 1)

Creating Visualizations

Before we can move forward with the Closeread extension, we will want to create the basic graph that we will work off of. Create a scatterplot with Year on the x-axis and Hotdogs_per_minute on the y-axis. Add a title and fix the labels. Here is the sample code if needed:

Click to reveal code 
ggplot(data = hotdog_men, aes(x = Year, y = Hotdogs_per_minute)) +
  geom_point() +
  ylim(0, 8) + ## this line ensures the dimensions are the same for all plots
  labs(y = "Hotdogs Per Minute", 
       title = "Hotdogs Eaten by Competition Winner By Year") +
  theme_minimal()

Placing a graph as a sticky

We want to insert the graph we have made as a sticky on the right of the document so that we can have associated scrolly text on the left. To do so, you will want to name your graph with ::: {#cr-graphname}. The first graph in the example is named @cr-biggraph.

Uncomment any commented lines when you add to your document.

::: {#cr-biggraph}
## ```{r}
ggplot(data = hotdog_men, aes(x = Year, y = Hotdogs_per_minute)) +
  geom_point() +
  ylim(0, 8) +
  labs(y = "Hotdogs Per Minute",
       title = "Hotdogs Eaten by Competition Winner By Year") +
  theme_minimal()
##```
:::

Once you have your graph named as a sticky, it will not show up until you call it. Add the text that you want to go along with your plot with @cr-biggraph following your text to assign it to your plot.

Here is the example code, but feel free to write what you want.

There are a few interesting trends that stand out here. @cr-biggraph

Why do we think that there is a continual decrease in the first \~10 years?

Notice that any additional paragraphs of text do not need to call the graph if they are directly following a paragraph that calls it. The graph will not disappear unless you call something else or tell it to go away.

Runtime

We want to talk about the trend at the beginning of the graph now. Looking at the data and reading about the contest from the Wikipedia page, you will see that earlier competitions had much shorter runtimes than more recent competitions.

Alter the original graph to highlight the first 8 years of the competition and label the points with how many minutes the total competition was.

Click to reveal code 
## uncomment lines for r chunk
::: {#cr-runtime}
## ```{r echo=FALSE, message=FALSE, warning=FALSE}

hotdog_time <- hotdog_men |> slice(44:51) 

ggplot(data = hotdog_men, aes(x = Year, y = Hotdogs_per_minute)) +
  geom_point() +
  ylim(0, 8) +
  labs(y = "Hotdogs Per Minute", title = "Hotdogs Eaten by Competition Winner By Year") +
  geom_text_repel(data = hotdog_time, aes(label = Time)) +
  geom_point(data = hotdog_time, size = 2, colour = "red") +
  theme_minimal()
## ```
:::

Add some text to explain the context of your graph to readers and call the graph to go along with it.

This means that contestants were able to eat more hotdogs per minute as they 
were eating for less time. @cr-runtime

2001 Jump

We also want to explain the large jump in number of hotdogs eaten between 2000 and 2001. If you have not already found the explanation for this jump, continue reading the Wikipedia page.

Once you have a general understanding to be able to continue your storytelling, alter the original graph to highlight the winner in 2001 and label with the name of the winner. Use geom_text_repel to move the name of the winner for aesthetic purposes. These coordinates have been found to work best.

nudge_y = -0.75, nudge_x = -1
Click to reveal full code 
hotdog_2001 <- hotdog_men |> slice(25) |>
  summarise(Year, Winner, Hotdogs_per_minute)

ggplot(data = hotdog_men, aes(x = Year, y = Hotdogs_per_minute)) +
  geom_point() +
  ylim(0, 8) +
  labs(y = "Hotdogs Per Minute", title = "Hotdogs Eaten by Competition Winner By Year") +
  geom_text_repel(data = hotdog_2001, aes(label = Winner),
                  nudge_y = -0.75, nudge_x = -1) +
  geom_point(data = hotdog_2001, size = 2, colour = "red") +
  theme_minimal()

Kobayashi to Chestnut

As you have most likely learned from your reading or previous knowledge, after Kobyashi reigned for many years, he was overtaken in 2007 by Joey Chestnut.

Alter the original graph to make a visualization showing the years in which Kobayashi won in one color and the years Chestnut won in another. Label both with the name of the winner. Add a text section to go with the plot.

Click to reveal code 
hotdog_kobayashi <- hotdog_men |> filter(Winner == "Takeru Kobayashi")
hotdog_joey <- hotdog_men |> filter(Winner == "Joey Chestnut")
hotdog_kobayashi_label <- hotdog_kobayashi |> slice(4)
hotdog_joey_label <- hotdog_joey |> slice(15)

ggplot(data = hotdog_men, aes(x = Year, y = Hotdogs_per_minute)) +
  geom_point() +
  ylim(0, 8) +
  labs(y = "Hotdogs Per Minute", 
       title = "Hotdogs Eaten by Competition Winner By Year") +
  geom_text_repel(data = hotdog_kobayashi_label, aes(label = Winner), 
                  nudge_y = -0.5, colour = "red") +
  geom_point(data = hotdog_kobayashi, size = 2, colour = "red") +
  geom_text_repel(data = hotdog_joey_label, aes(label = Winner),
                  nudge_y = 0.5, colour = "dodgerblue") +
  geom_point(data = hotdog_joey, size = 2, colour = "dodgerblue") +
  theme_minimal()
