Introduction

The Nathan’s Famous Fourth of July International Hot Dog Eating Contest has a 100+ year tradition. Over time, competitors have pushed the limits of human eating ability in ways no one might have imagined.

This module explains the process of creating effective data visualization plots using tidyverse and the closeread HTML Quarto extension.

Background

Closeread is a custom format for Quarto that enables scrollytelling features for HTML documents.

Please, refer to the link above to download and install the extension before continuing along with the module.
For additional, detailed guidance on the use of Closeread, please watch this talk
“Andrew Bray - Closeread: bringing Scrollytelling to Quarto” at the Posit Conference 2024.
Scrollytelling

Scrollytelling is a style of web design that transitions graphics and text as a user scrolls, allowing stories to progress naturally. Despite its power, scrollytelling typically requires specialist web dev skills beyond the reach of many data scientists.

Closeread is a Quarto extension that makes a wide range of scrollytelling techniques available to authors without traditional web dev experience, with support for cross-fading plots, graphics and other chunk output alongside narrative content. You can zoom in on poems, prose and images, as well as highlighting important phrases of text.

How to create a section

  • Section: Elements within a section will appear as a closeread story while elements outside of a section will appear as a normal Quarto HTML document.

Creating sections follows the syntax ::::{.cr-section_name} where section_name is the name that will identify the section whenever it needs to be referenced. The three consecutive colons go in front of the section notation and at the end of the section as :::: on its own line of code to act as delimitation for the space of the section.

How to create a sticky

  • Sticky: An element of your document that you wish to do a close read of is called a “sticky”; as the user scrolls, it will remained pinned in the middle of the main column like a sticky note.

Creating stickies (on a very general sense) involves de syntax :::{#cr-sticky_name} where sticky_name is the name that will identify the sticky whenever it needs to be referenced. The three consecutive colons go in front of the section notation and at the end of the section as ::: on their own line of code to act as delimitation for the space of the sticky.

Section on referencing and other details of the closeread code style.

Referencing is done within the Quarto code to trigger a sticky to appear as soon as the line of the reference reaches the middle of the screen. Use the syntax \(@cr-sticky_name\) to trigger a sticky to show on the closeread section.

Learning Objectives

By the end of this activity, you will be able to:

  1. Create effective data visualization plots from the data through contextual analysis.

  2. Analyze and describe the trends and significant features of the data set with appropriate annotations.

  3. Use the closeread HTML extension for Quarto documents to create a scrollytelling file.

  4. Interpret general trend information of a specific data set from the visualization plots and annotations.

Methods

Technology requirement: The activity handout provides a guidance Quarto document and a solved HTML, for reference on solutions and code references.

  1. Plots: Familiarity with tidyverse, mainly ggplot2, dplyr and tidyr as a techniques for statistical analysis and data representation is crucial. You should understand how to use R code to perform changes on the data set.

  2. Basic understanding of the closeread syntax to create sections, stickies and reference files within the code to appear as desired in the final HTML document.

Data

A data frame for 69 champions of the Nathan’s Famous Hot Dog eating competition from 1972 until 2025. The data has been scraped from the web source Wikipedia, “Nathan’s Famous Hot Dog Eating Contest”, and cleaned with the R tidyverse library. The variable for the dogs eaten is the result of calculating the ratio of the hot dogs each winner ate in non-extra time periods by the time of the competition in minutes. This was done because the competition times are not the same for all the years in the data set.

Download data: hotdog_clean.csv

Variable Descriptions
Variable Description
Year Year the competition took place
Sex Sex of the winner for the year the competition took place
Dogs Dogs eaten per minute
Winner Name of the winner for their sex category
Time Time in minutes the competition elapsed
